jstd-env

Infrastructure for fast and easy creating and running tests with jsTestDriver

Why should i use this?

Jstd-env is open-source project directed to simplify work with JsTestDriver when writing javascript code. It can:

  • run your project tests from any directory you want
  • complete testcases names
  • stop/start server for you
  • remember what port was used last time
  • ...

Installation

The shortest way is 

sudo curl https://github.com/kunik/jstd-env/raw/master/install.sh | sh

But the recommended way is to install jstd-env under your home directory: 

export _SBIN_DIR="$HOME/sbin"
export _BIN_DIR="$HOME/bin"
curl https://github.com/kunik/jstd-env/raw/master/install.sh | sh

By running first two commands you sets global variables _SBIN_DIR and _BIN_DIR in your environment. If they are set before running the installation script it will try to install needed files to directories they represent.

Of course both of directories should exist and be writable. Also, _BIN_DIR should be in your PATH variable.

Manual installation

You can also clone the project with Git by running: 

git clone git://github.com/kunik/jstd-env

Then you should create two symbolic links in folder which is in your PATH: 

cd jstd-env
ln -s "$PWD/jstd.sh" ~/bin/jstd
ln -s "$PWD/jstd_stuff ~/bin/jstd_stuff

After that you need to download JsTestDriver.jar into your jstd_stuff dir: 

cd jstd_stuff
wget http://js-test-driver.googlecode.com/files/JsTestDriver-1.3.1.jar

Using

To start using jstd-env and JsTestDriver you just need to enter your js project directory and run: 

jstd start

This command will start JsTestDriver server for you. It is also possible to specify port after start command.

JsTestDriver config file is located it .jstd folder and called config.yaml by default.

After editing files you can run all the tests: 

run

Or any test case you like 

run [tab]ob[tab]
observer object
run obs[tab]erver
